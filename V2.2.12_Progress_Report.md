## ❌ 待修正事項 (Critical Issues)

### 1. 翻譯卡片標籤異常
*   **現象**：左上角的模型標示（Gemini/Gemma）無法正確顯示。
*   **原因初探**：`result.js` 在接收 `appendResult` 訊息時，資料結構中的 `usedModelName` 可能未正確傳遞或解析。
*   **修正方案**：檢查 `background/index.js` 在發送訊息時是否包含模型名稱，並修正 `result.js` 的判斷邏輯。

### 2. 跳轉後翻譯未啟動
*   **現象**：在結果頁點擊「下一話」雖然能跳轉網頁，但跳轉後不會自動觸發 `crawlImages` 與後續翻譯。
*   **原因初探**：`navigateAndTranslate` 邏輯在背景腳本中可能因為分頁 ID 變更或狀態丟失而失效。
*   **修正方案**：重寫跳轉後的監聽邏輯，確保新頁面載入完成後能自動發起 `handleStartMangaBatchPCMode`。

### 3. 全域邏輯漏洞檢視
*   **重點**：
    *   **狀態同步**：Service Worker 重啟後，正在進行的任務狀態是否會丟失？
    *   **逾時處理**：當圖片抓取失敗或 API 永久卡死時，UI 是否會一直卡在「翻譯中」？
    *   **多 Key 併行度**：目前的備援並行翻譯是否真的有發揮多 Key 優勢？

### 4. 行動版翻譯按鈕失蹤 (Critical)
*   **現象**：行動控制台下方的「開始翻譯」按鈕無法顯示或點選。
*   **原因初探**：`main.js` 讀取圖片屬性名稱錯誤（`url` vs `src`），導致圖片無法選取，按鈕保持 `disabled`；且 Flex 佈局在部分手機螢幕下可能溢出。
*   **修正方案**：修正屬性名稱，並在 `mobile/index.html` 中增加對不同螢幕尺寸的佈局適配。
