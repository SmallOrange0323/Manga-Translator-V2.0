# Manga Translator V2.0: 語境感知翻譯系統 (Context-Aware Translation System) 提案

## 1. 背景與問題診斷
目前的翻譯系統主要依賴單一圖片的視覺分析與簡單的「關鍵字對應」語彙庫。儘管能固定特定名詞的譯名，但仍普遍存在以下問題：
- **身分關係混淆**：將同事/上司誤判為同學，導致稱呼（如：XX桑 -> XX同學）錯誤。
- **人稱代名詞錯誤**：無法從單一畫面判斷性別，導致「他/她」混用。
- **語氣不一致**：無法判斷長幼尊卑，導致敬語與平語（口語）切換混亂。

## 2. 核心構想：故事設定集 (Story Bible)
模仿漫畫單行本前言的模式，在翻譯流程中導入「全域背景」與「人物誌」，作為 AI 翻譯時的強制約束條件。

## 3. 功能模組設計

### A. 全域故事設定 (Global Story Context)
- **UI 設計**：在側邊欄或設定頁面提供一個長文字輸入框。
- **內容範例**：`這是一部關於科技公司內部競爭的職場劇。主角「佐藤」是新進職員，語氣謙卑；「田中」是其主管，語氣嚴厲且常帶有專業術語。`
- **技術運作**：將此段文字直接注入到 Gemini 的 `System Prompt` 頂層。

### B. 人物誌語彙庫 (Character Profiles)
- **UI 設計**：擴充現有的語彙庫介面，增加「特徵描述」欄位。
- **資料欄位**：
  - `Original Name`: 原文 (e.g., 佐藤)
  - `Translation`: 譯文 (e.g., 佐藤)
  - `Traits/Bio`: 人設 (e.g., 男性，25歲，性格膽小，對田中語氣極度尊敬)
- **技術運作**：翻譯時自動比對圖片中出現的文字，若符合語彙庫條目，則將相關 `Traits` 作為「背景知識」傳送給 AI。

### C. 語氣約束系統 (Tone & Honorifics System)
- **自動偵測**：強制 AI 分析日文原始後綴（-san, -sama, -kun, -senpai）。
- **對應規則**：定義一套對應表，例如：
  - `-san` (職場情境) -> 先生/小姐 或 職稱。
  - `-senpai` -> 前輩。
  - 禁止將非校園情境的稱呼預設為「同學」。

## 4. 技術實作路徑 (Roadmap)

### 第一階段：靜態上下文注入 (Static Injection)
- 實作「全域背景」輸入框。
- 調整 `translate-api.js`，將背景資訊拼接到 Prompt 請求中。

### 第二階段：結構化人物誌 (Structured Bio)
- 更新 `glossary-manager.js` 儲存格式。
- 增加 UI 欄位讓使用者輸入角色屬性。

### 第三階段：AI 自動引導 (AI-Guided Context)
- 開發「一鍵分析人設」功能：由 AI 先掃描漫畫的前幾頁，自動產生推薦的故事背景與人物誌草稿，由使用者確認後存入系統。

## 5. 討論重點 (給其他 AI 的引導問題)
1. **傳輸量優化**：過長的人物誌是否會導致 API 消耗過快或超出 Context Window？
2. **多模態結合**：如何更精準地引導 Gemini 結合「人設描述」與「圖片視覺特徵（服裝、表情）」？
3. **動態更新**：如何處理故事進行中角色關係的變化（例如從同事變成夫妻）？
